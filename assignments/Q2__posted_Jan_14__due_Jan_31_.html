2 Q2 (posted Jan 14, due Jan 31)

Language: Racket.

Weight: 3.0

In this question, you will implement a lambda-calculus stepper. The starter file contains some support code which is trivial but whose use will make your code more readable, and skeletons of a number of functions for you to complete.

Racket has a menu item (and a keybinding) to insert a \(\lambda\) into a program. This can be substituted for the keyword lambda, but we will use it in a more mundane fashion, in a list representing a term of the lambda calculus. An abstraction will be a list of length 3. For example, '(λ x x) is the representation of \(\lambda x.x\). An application is a list of length 2. For example, '((x y) z) is the representation of \(x~y~z\).

The Marmoset tests involve the following functions:

  • alpha-equiv consumes two lambda-calculus expressions and produces a Boolean which is true iff the two expressions are \(\alpha\)-equivalent. This function is used mainly for testing.

  • subst consumes a symbol (representing a variable \(x\)) and two expressions (representing terms \(N,M\)) and produces an expression (representing \([x \mapsto N]M\)).

  • normalize consumes an expression and produces an expression representing its normal form under normal order reduction.

In writing normalize, you may find the following handy:

  • reducible consumes an expression and produces a Boolean which is true iff the expression contains a redex.

  • reduce consumes an expression and produces the expression that results from one step of normal order reduction.

For bonus marks (about 20%), let’s distinguish the "fully-parenthesized" representation above from a more "human-readable" representation. We can use the conventions mentioned in lecture for omitting parentheses, plus allow multiple variables per lambda (so \(\lambda x . \lambda y .x\) becomes \(\lambda x y.x\)). We use : instead of a dot (which has a special meaning in Racket) and we may need to wrap a list around the very outside of an expression: \(\lambda x y. x\) becomes (λ x y : x).

  • parse consumes a list representing a human-readable representation of a lambda-calculus expression and produces a fully-parenthesized version.

  • format-lc consumes a fully-parenthesized representation of an expression and produces a string containing a human-readable representation. For example, applied to (λ x (λ y x)), it would produce "λ x y : x" (nine characters long).

Do not use mutation, with the following exception: you may use the built-in function gensym, which is not pure (two identical expressions using it will produce different values).
