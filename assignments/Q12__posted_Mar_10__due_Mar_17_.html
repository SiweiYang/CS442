12 Q12 (posted Mar 10, due Mar 17)

Language: Haskell. In these and subsequent questions, you may use the modules Data.List and Data.Char. Do not include main unless specifically asked to by the question.

Weight: 1.0

The type [a] is an instance of the type class Ord if the type a is; in this case, the ordering is lexicographic. This is the standard English "dictionary order" for characters. Another useful ordering is length-lexicographic, in which strings (or, more generally, lists of ordered type) are compared first according to length (if one is shorter, it wins), and then according to lexicographic order if the lengths are equal.

Recall from CS 241 that a regular expression is built up from base terms that are symbols in some fixed alphabet \(\Sigma\), plus operators \(+\) (alternation), \(\cdot\) (concatenation), and \({}^*\) (repetition or Kleene star). We can give a recursive definition of the language \([[{e}]]\) of strings over \(\Sigma\) denoted by a regular expression \(e\). \(\epsilon\) denotes the empty string.

\begin{align*}[[\phi]] &= \phi\\ [[\epsilon]] &= \{\epsilon\}\\ [[{a}]] &= \{a\}\text{ for } a \in \Sigma\\ [[{e_1+e_2}]] &= [[{e_1}]] \cup [[{e_2}]]\\ [[{e_1\cdot e_2}]] &= \{uv \mid u \in [[{e_1}]], v \in [[{e_2}]]\}\\ [[{e_1^*}]] &= \{\epsilon\} \cup [[{e_1 \cdot e_1^*}]]\end{align*}

We can describe regular expressions with symbol alphabet type a using the following Haskell datatype:

data RE a = Empty | Epsilon

          | Symbol a | Alt (RE a) (RE a)

          | Concat (RE a) (RE a) | Repeat (RE a)

          deriving (Show, Eq, Ord)

Write a function reGen :: (Ord a) => RE a -> [[a]] that produces the list of all strings over a described by its argument, in length-lexicographic order. Here is a sample interaction using the regular expression \((a+b)^*\).

take 10 $ reGen $ Repeat $ Alt (Symbol 'a') (Symbol 'b')

> ["","a","b","aa","ab","ba","bb","aaa","aab","aba"]

reGen should use structural recursion. Do not use a sorting algorithm; it isn’t necessary.
