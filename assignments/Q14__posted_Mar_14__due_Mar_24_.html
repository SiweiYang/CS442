14 Q14 (posted Mar 14, due Mar 24)

Language: Haskell.

Weight: 1.0

Recall from CS 241 that deterministic finite automata (DFAs) are state machines operating on strings viewed as sequences of symbols or alphabetic characters. We can describe DFAs with symbol alphabet type a using the following datatype:

data DFA a = DFA [(Int, a, Int)] Int [Int]

             deriving (Show, Eq, Ord)

The states of a DFA are positive integers. DFA trs init fins describes a DFA with initial state init, list of final states fins, and transition list trs. Each transition specifies a "from" state, a symbol on which the transition takes place, and a "to" state. This single-step transition function can be naturally extended to a multi-step transition function on a string (list of symbols). The machine accepts a string if and only if the multi-step transition function applied to the initial state and the string produces a final state.

You may assume that an instance of a DFA is actually deterministic (at most one transition for each "from" state and symbol, that is, the single-step transition function really is a partial function); as a warmup exercise (not to be handed in) write a Haskell function to check this.

Write a function dfaGen :: (Ord a) => DFA a -> [[a]] that produces the list of all strings accepted by its argument, in length-lexicographic order. There is no reason to use code from Q13; the technique here is different.

A sorting algorithm is useful here, and you may apply sort from Data.List, but at most once per application of dfaGen, and not to the strings being generated.
